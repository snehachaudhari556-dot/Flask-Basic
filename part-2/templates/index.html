What happens when you open this page?
  1. Your browser sends a request to the Flask server
  2. Flask matches the URL (/) with a route
  3. The home() function is executed
  4. index.html is loaded from the templates folder
  5. The final HTML is sent back to your browser

Project Overview

This project demonstrates how Flask connects Python code with HTML templates to create clean, structured web pages. It is designed for beginners who want to understand backend fundamentals before moving to advanced features.

Learning Goal:

Understand how Flask routes, templates, and requests work together to generate dynamic web pages.

Why Flask Uses Templates?

  • Separation of Concerns: Python handles logic, HTML handles presentation
  • Easy Maintenance: UI can be updated without changing backend code
  • Reusability: Same template can display different data
  • Jinja2 Power: Supports variables, loops, and conditions

Best Practices for Beginners

  • Always store HTML files inside the templates/ folder
  • Keep route names simple and meaningful
  • Use template inheritance to avoid repetition
  • Test each route after creating it
